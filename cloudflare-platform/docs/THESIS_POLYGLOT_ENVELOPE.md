# THESIS Polyglot Builder Envelope

## Doctrine

The product is not allegiance to one chain, cloud, or language. It is the builder's mastery of a governed polyglot runtime. THESIS accepts a mission, selects the runtime that best satisfies latency, sovereignty, numerical performance, portability, and enforcement, then returns one coherent result with lineage.

## Runtime ownership

| Plane | Canonical owner | Responsibility |
|---|---|---|
| Builder intelligence | THESIS | mission decomposition, capability routing, dependency graph, receipts |
| Local mind | HIM browser brain | Transformers.js/WebGPU inference, embeddings, memory, documents, wallet UI |
| Orchestration | NIO/Python | FastAPI engines, research, indexing, security, long-running workers |
| Quantitative compute | MATHESIS/Julia | spectral methods, Monte Carlo, VaR/CVaR, optimization, gas models |
| Portable execution | Node/WASM | bounded cross-platform functions |
| Edge envelope | Cloudflare/HERMES | UI, durable sessions, queues, schedules, service bindings, control plane |
| Enforcement | CPL + Solidity | PolicyKernel, ReceiptChain, LawBook, SovereignVault |
| Settlement | Monad | EVM state, receipts and owner-approved transactions |

## User flow

1. Describe — natural-language mission, file, repository, or contract objective.
2. Map — THESIS displays the runtime plan and why each runtime was selected.
3. Simulate — read chain state, estimate gas, run Python/Julia/local-model analysis.
4. Build — produce code, documents, tests, manifests, and deployment candidates.
5. Govern — CPL evaluates permissions, blast radius, dependencies, and approval.
6. Sign — wallet presents an unsigned transaction; only the owner signs.
7. Verify — Monad receipt plus THESIS hash lineage closes the mission.
8. Package — downloadable Markdown/CSV/JSON/ZIP deliverable.

## Setup is a product surface

Users choose one or more capability lanes:

- Browser-only: local HIM, memory, documents, WASM, and wallet. No cloud secret.
- Session Cloudflare token: pasted into Settings, held only in session storage, sent over the authenticated same-origin channel, never persisted.
- Managed Worker secret: configured during Cloudflare deployment for a durable hosted workspace.
- Python/Julia bridge: optional governed service bindings; unavailable lanes remain visible but disabled.
- Monad: public read RPC plus browser wallet for explicit signing.

Wrangler commands remain an operator/CI option, not the primary onboarding.

## Non-negotiable laws

- No private key enters a Worker, model prompt, Durable Object, log, or receipt.
- No model may sign or silently broadcast.
- Read authorization and execution authorization are distinct.
- Every mutation is proposal-bound, expiring, receipted, and scope checked.
- HIM weights means the repository-native checkpoint. Workers AI is an explicitly labeled optional provider, never a hidden fallback.
- Monad smart contracts remain EVM bytecode; polyglot execution happens around the chain, not inside it.
